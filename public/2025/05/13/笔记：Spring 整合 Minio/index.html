一、理论

Minio 配置

[!NOTE] 注意事项

  1. MinioClient 用于与 MinIO 服务建立连接，发送请求。而所谓的 Minio 配置，就算将 MinioClient 声明为一个 Bean，使用时直接注入即可
  2. MinioClient 底层基于 OkHttpClient，使用的是 HTTP 连接池机制，这与我们日常理解的 MySQL 等数据库连接池有所不同，所以 MinioClient 是线程安全的，但是高并发时也会存在瓶颈
